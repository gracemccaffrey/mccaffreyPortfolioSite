HealthHub

A health literacy and awareness platform that helps underserved communities find care and understand healthcare basics.

Problem Statement

Many individuals face barriers in understanding healthcare options, navigating insurance, and locating nearby providers. These challenges are especially prominent in underserved areas, where provider shortages and unreliable internet access can delay treatment and worsen outcomes.

Limited provider access

Long travel distances and shortages of care options.

Low health literacy

Medical terms and care processes can be confusing.

Insurance & cost barriers

Uncertainty around coverage and affordability.

Connectivity constraints

Limited broadband reduces access to online resources.

Challenges

  • Limited access to nearby providers
  • Medical terminology confusion
  • Financial & insurance barriers
  • Poor broadband access in rural regions

Research & Context

Social & Cultural Context

Rural communities can have lower trust in external systems, and some populations have limited technology adoption. Designing for clarity and trust was essential.

Physical Constraints

Poor connectivity and device limitations meant we prioritized lightweight pages, mobile-first UX, and offline fallbacks.

Health Literacy Gaps

Many users struggle to interpret health information. We focused on plain-language definitions, FAQs, and predictable navigation patterns.

Our Solution

Interactive Provider Map

Search providers by specialty and ZIP code, explore results on a map, and view provider details in expandable cards.

Health Literacy Tools

Terminology and FAQs help users understand common healthcare concepts, insurance basics, and what to expect at appointments.

Offline Fallback Dataset

When internet is limited, users can still search a curated local provider dataset by ZIP code.

Mobile-First, Accessible UI

Designed for small screens and low-bandwidth environments with clear hierarchy and readable typography.

Key Design Decisions

Platform Choice: PWA Website

Chose a website-style progressive web app for broad accessibility (no installation) and easier maintenance.

Maps: Google Maps API

Selected for geospatial accuracy, documentation quality, and reliable routing/navigation support.

Offline Strategy: Curated Dataset

A full-state dataset was too large for fast loading. We implemented a Central Ohio offline dataset to balance performance and access.

Why it matters

These decisions prioritize usability in real-world constraints: limited connectivity, older devices, and users who need simple, trustworthy information quickly.

Technical Architecture

Client (Browser / PWA) → UI + Filtering
NPPES API (CMS) → Provider Data
Google Maps API → Geolocation + Map
Local Fallback JSON → Offline Search

Data is filtered, deduplicated, and limited to keep results readable and performant.

Tech Stack

  • Frontend: HTML, CSS, JavaScript
  • Data: NPPES API + curated JSON fallback
  • Mapping: Google Maps API
  • Patterns: modular content injection, filtering, dedupe

Engineering Highlights

The NPPES API returns many entries outside the scope of “providers users would realistically visit.” We filtered results using a curated set of taxonomy codes for relevant specialties.

To reduce clutter, we merged entries representing the same organization by normalizing names and grouping providers within a small geographic radius.

Results were limited (e.g., within a reasonable radius and capped count) to keep the UI fast and prevent information overload—especially important in low-bandwidth settings.

FAQ and terminology content is stored in structured data files and injected dynamically, so content updates don’t require editing HTML.

Sustainability & Ethics

Environmental Considerations

HealthHub was designed to be lightweight and reduce reliance on continuous server calls. Offline support helps reduce infrastructure load while improving access.

  • Local JSON fallback reduces repeated network requests
  • Designed to run on older devices (lower hardware demands)
  • Open-source viability supports long-term sustainability

Ethical Design

The platform is intended as a free resource—no paywalls and no advertising—focused on empowerment, clarity, and accessibility.

  • Plain-language content to support varied literacy levels
  • UX choices aimed at trust and predictable navigation
  • Offline-first thinking for digital equity

Reflection & Future Work

What I’d improve next

  • Expand offline coverage beyond Central Ohio
  • Add multilingual support for broader accessibility
  • Enhance specialty filtering and care navigation guidance
  • Explore partnerships to improve data freshness and outreach

Key takeaway

Research-informed design + practical implementation tradeoffs led to a solution that stays usable under real-world constraints.
